**Proof Phases:**
* Phase 1: Notation and Formalization. Begin by establishing a consistent set of mathematical notation. Define the state space, the cost metric (e.g., Euclidean, Manhattan), and the specific objective function (e.g., minimizing path length, maximizing information gain). Explicitly state the definition of the Optimal Solution ($OPT$) and the Algorithmic Solution ($ALG$). If the problem involves specific structures such as submodularity, monotonicity, or metric space properties (triangle inequality), explicitly define these properties in the context of the given problem.
* Phase 2: Establishing Lower and Upper Bounds. Derive the bounds necessary for the approximation proof.
    1. Lower Bound on OPT: Determine a mathematical lower bound for the cost of the optimal solution. In many path planning problems, this might involve the Minimum Spanning Tree (MST), the diameter of the set, or area coverage bounds.
    2. Upper Bound on ALG: Analyze the worst-case behavior of the provided algorithm. Express the cost of $ALG$ in terms of the components used to bound $OPT$.
* Phase 3: Derivation of the Approximation Ratio. Using the bounds established in Phase 2, derive the ratio $\alpha$ such that $Cost(ALG) \leq \alpha \cdot Cost(OPT)$ (for minimization) or $Reward(ALG) \geq \frac{1}{\alpha} \cdot Reward(OPT)$ (for maximization).
    1. Clearly state every lemma or theorem used (e.g., "Using the Triangle Inequality...", "By the submodularity property of the objective function...").
    2. Show step-by-step algebraic manipulation.
    3. Identify if the approximation is constant, logarithmic, or polynomial with respect to the input size $n$.
